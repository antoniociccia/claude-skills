v1.0 — Development Framework for Claude Code

Structured development
for AI-assisted coding

A configurable 12-step gate process with 4 project profiles, 6 step skills, and 6 specialized agents. Drop it into any project. Pick a profile. Claude follows the process.

# CLAUDE.md active_profile: porting # That's it. Claude reads the profile # and enforces the gate process.

Profiles

Four profiles, one framework

Each profile activates a subset of the 12 steps. Click a profile to see its pipeline.

porting
Migrating from one stack to another. Legacy analysis, DB parity, API parity.

greenfield
Building new features from scratch. YAGNI, interface-first, incremental delivery.
11 / 12 steps active
bugfix

6 / 12 steps active

Skills

Step skills

Invoked automatically at the appropriate step. Each skill contains templates, checklists, and guidelines.

gate-process
Master controller
Reads the active profile, enforces the step sequence, manages approval gates. The orchestrator.
functional-doc
Step 2
Generates business requirement documents: rationale, actors, input/output, business rules, integration points.
workflow-doc
Step 3
Generates step-by-step operational flows: API calls, DB state changes, error scenarios, edge cases.
parity-check
Step 9
Dual-endpoint comparison. Response shape (keys + types, not values) and DB state after writes.
security-scan
Step 12
10-item security checklist: input validation, auth, injection, data exposure, dependencies, audit trail.
manual-writing
Step 11
Templates for developer manuals (API reference) and user manuals (plain language workflows).
behaviour-test
Step 10
E2E test guidelines: what to verify, anti-flaky principles, test structure, environment config.
Agents

Specialized sub-agents

Delegated tasks. Each agent detects the project's stack and adapts. Stack-agnostic by design.

test-runner
sonnet
Detect framework, run tests, classify failures (TEST_BUG, CODE_BUG, ENV_ISSUE, FLAKY), fix broken tests.
manual-writer
haiku
Analyze completed code, generate developer manual (API reference) and user manual (plain language guide).
legacy-analyzer
sonnet
Analyze any legacy codebase in any language. Extract API contracts, business rules, DB operations.
parity-tester
sonnet
Generate dual-endpoint parity test scripts. Compare response shape and DB state between v1 and v2.
swagger-generator
sonnet
Detect API framework (NestJS, Express, Flask, Spring, Go), generate complete OpenAPI annotations.
security-scanner
sonnet
Framework-aware static analysis. 10 security checks with severity ratings and actionable recommendations.

The Iron Rule
No block of work is complete until ALL active steps for the profile are done. If a step is active and not done, the work is INCOMPLETE. Do not move to the next block until all steps for the current block are verified. Gates are rigid. When a step is active, its gate cannot be skipped.
